# Path Traversal атаки в Node.js уеб приложения

Курсов проект по дисциплината **Основи на сигурното уеб програмиране**.

**Изготвил:** Калоян Механджийски  
**Факултетен номер:** 5MI0700025  
**Специалност:** Информационни системи  
**Технология:** Node.js + Express

## 1. Цел на проекта

Проектът демонстрира как възниква Path Traversal уязвимост при неправилна обработка на потребителски вход, използван като файлов път.

Приложението съдържа две версии на endpoint за сваляне на файлове:

- `/download-vulnerable` — уязвима имплементация;
- `/download-secure` — защитена имплементация.

## 2. Инсталация

```bash
npm install
```

## 3. Стартиране

```bash
npm start
```

След това отвори:

```text
http://localhost:3000
```

## 4. Уязвима версия

Нормална заявка:

```text
http://localhost:3000/download-vulnerable?file=hello.txt
```

Опит за Path Traversal:

```text
http://localhost:3000/download-vulnerable?file=../private/secret.txt
```

При уязвимата версия приложението може да върне файл извън разрешената директория.

## 5. Защитена версия

Нормална заявка:

```text
http://localhost:3000/download-secure?file=hello.txt
```

Опит за Path Traversal:

```text
http://localhost:3000/download-secure?file=../private/secret.txt
```

Този опит трябва да върне:

```text
403 Access denied.
```

## 6. Как работи защитата

Защитеният endpoint използва:

- `path.normalize()` за нормализиране на подадения път;
- `path.resolve()` за изчисляване на абсолютен път;
- проверка дали крайният път започва с позволената директория `BASE_DIR`;
- HTTP 403 при опит за достъп извън позволената директория.

## 7. Основна идея

Потребителят не трябва директно да контролира пътя до файл в сървърната файлова система.

Правилният подход е приложението да действа като посредник, който валидира всяка заявка и позволява достъп само до предварително разрешена директория.
